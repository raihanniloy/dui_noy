# 29 Card Game — Plan 2 Design: Bots + Simulator

Date: 2026-06-15
Status: Approved by user

## Goal

Heuristic bots that play 29 by consuming the same `PlayerView` + `legalActions` contract as a future human/network client, plus a bot-vs-bot invariant simulator. Zero Phaser/browser dependencies, deterministic with seeded RNG.

**Builds on:** Plan 1 engine (`docs/superpowers/plans/2026-06-12-29-engine.md`). Engine is complete: `applyAction`, `legalActions`, `playerView`, 49 tests green.

**Model:** Opus 4.8 (strategy reasoning).

## Locked decisions

1. **Card-counting heuristic** — bots track public information (cards played per suit, revealed voids, trump-out) and use it in move scoring. Not a simple greedy bot.
2. **Easy + Hard difficulty** in this plan. Difficulty = additive noise on move scores; `hard` = argmax, `easy` = argmax over score + `noise·rand`.
3. **Simulator is a Vitest test** — default 200 games, `SIM_GAMES` env var raises to 1000. Stays inside `npx vitest run`.

## Architecture

```
PlayerView + legalActions (from engine, Plan 1)
        │ (read-only contract)
CardMemory      ← public info accumulated across a hand
        │
heuristics.ts   ← pure scoring functions (bid / trump / play)
        │
bot.ts          ← decideAction(view, legal, memory, difficulty): Action
```

Bots never see other hands or concealed trump suit — only what `playerView` exposes. Same contract a websocket client would get.

## Modules

### `src/bots/cardMemory.ts`

Public-information tracker for one hand, updated from engine `GameEvent`s.

```ts
export interface CardMemory {
  readonly played: ReadonlySet<string>;        // "suit-rank" of every card played this hand
  readonly playedBySuit: Readonly<Record<Suit, number>>;  // count played per suit
  readonly voids: Readonly<Record<Seat, ReadonlySet<Suit>>>; // suits a seat has shown void in
  readonly trumpRevealed: boolean;
}
export function emptyMemory(): CardMemory;
// ledSuit = suit led in the current trick (null if this play is the lead).
export function observe(mem: CardMemory, ev: GameEvent, ledSuit: Suit | null): CardMemory;
```

- `CardPlayed` → add to `played`/`playedBySuit`. If `ledSuit !== null` and the card's suit ≠ `ledSuit`, the player is void in led suit → record in `voids`.
- `TrumpRevealed` → set `trumpRevealed`.
- `HandStarted`/`HandScored` → reset for next hand (caller starts fresh via `emptyMemory`).

Derived helpers (pure, on memory + view):
- `isBoss(card, ledSuit, mem, trumpSuit)` — no higher unplayed card of the relevant suit remains.
- `trumpsOut(mem, trumpSuit)` — count of trump cards already played.

### `src/bots/heuristics.ts`

Pure functions, no RNG inside (RNG/noise applied in `bot.ts`).

- `handStrength(hand): number` — weighted: J=4, 9=3, A=2 per control, plus suit-length bonus.
- `bidDecision(view): { value: number } | { pass: true }` — propose a bid from `handStrength` mapped onto 16–28 against current `legalActions` floor; pass when below threshold.
- `chooseTrumpMode(hand): TrumpMode` — longest+strongest suit → `suit`; balanced-weak hand → `seventh`; rarely `joker`.
- `doubleDecision(view): 'double' | 'redouble' | 'decline'` — double when opponent target looks unmakeable vs own strength; redouble when own hand is strong.
- `scorePlay(card, view, mem): number` — score a single legal play:
  - win the trick as cheaply as possible (lowest winning card),
  - if can't win, dump lowest point value; throw points only onto a trick the partner is currently winning,
  - conserve trump unless trumping wins meaningful points or is forced,
  - leading: prefer boss cards / lead from long strong suit.
- `revealDecision` / `marriageDecision` — reveal trump when void in led suit and holding trump helps; declare marriage when legal and target-shift favors own team.

### `src/bots/bot.ts`

```ts
export type Difficulty = 'easy' | 'hard';
export interface Bot {
  decideAction(view: PlayerView, legal: readonly Action[], mem: CardMemory): Action;
}
export function makeBot(difficulty: Difficulty, rng: Rng): Bot;
```

- Filters `legal` by action type for the current phase, scores candidates via `heuristics`, picks argmax.
- `easy`: add `NOISE · rng()` to each score before argmax (`NOISE` tuned so easy still avoids illegal/obviously-bad moves but misplays).
- Always returns an action present in `legal` (never illegal).

## Data flow (driver pattern, used by simulator and later GameClient)

1. `actingSeat(state)` → whose turn.
2. `playerView(state, seat)` + `legalActions(state, seat)`.
3. `bot.decideAction(view, legal, memory[seat])` → action.
4. `applyAction(state, action)` → new state + events.
5. Feed events into each seat's `CardMemory` via `observe`.
6. Repeat until `phase === 'done'`.

## Simulator — `tests/engine/simulate-bots.test.ts`

- 4 bots (configurable difficulty mix) play full games, seeded.
- Default `200` games; `SIM_GAMES` env var overrides (CI can run 1000).
- Invariants per game:
  - every completed hand: card points sum to 29, exactly 8 tricks (`TrickWon` count and points),
  - no `applyAction` ever returns `ok:false` for a bot-chosen action,
  - bidder-team score delta applied to bidder team only,
  - game terminates with `|score| ≥ WIN_TARGET`.
- Sanity: across all games, bidder team meets bid in > 25% of hands (bots not random). Loose lower bound only.

## Unit tests — `tests/bots/*.test.ts`

- `cardMemory.test.ts`: `observe` records plays, detects voids when off-suit played, sets `trumpRevealed`; `isBoss`/`trumpsOut` correct.
- `heuristics.test.ts`: `handStrength` monotonic (adding a J raises it); `scorePlay` picks the cheapest winning card and dumps lowest points when losing; `bidDecision` passes on a weak hand, bids on a strong one; `chooseTrumpMode` picks the long suit.
- `bot.test.ts`: `decideAction` output always ∈ `legal` across many random states; `hard` is deterministic for a given state; `easy` varies with rng.
