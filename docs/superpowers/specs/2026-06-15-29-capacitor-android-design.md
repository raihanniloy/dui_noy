# 29 — Plan 4: Capacitor Android Packaging (Design)

Date: 2026-06-15
Status: Approved (design)

## Goal

Wrap the existing Vite + Phaser web build as an installable **Android debug APK** for
sideloading. No store release, no signing, no iOS. The game logic, bots, and UI are
unchanged — this phase only adds the native shell and a thin platform layer.

## Why Capacitor

- Decided in the original card-game design: "web + Android from one codebase."
- Vite already builds with `base: './'`, so all asset/script paths are relative and load
  correctly from the `file://` WebView Capacitor uses. No path rewrite needed.
- Phaser 4 runs unmodified inside the Android System WebView.

## Scope

In:
- Capacitor project + Android platform, producing a working **debug APK**.
- Lock **portrait** orientation (game is 720×1280 portrait).
- Hardware **back button** handling.
- **Immersive fullscreen** (hide status/navigation bars).
- Custom app **icon + splash** from existing game art.
- A **local-environment setup check** (Android SDK, JAVA_HOME, etc.) so the build step
  fails fast with a clear message rather than a cryptic Gradle error.

Out:
- Signing, release builds, Play Store upload, versioning.
- iOS.
- Push notifications, analytics, deep links, in-app updates.

## Architecture

### Dependencies (devDependencies / dependencies)
- `@capacitor/core`, `@capacitor/cli`, `@capacitor/android`
- `@capacitor/screen-orientation` — portrait lock
- `@capacitor/app` — hardware back button + exit
- `@capacitor/status-bar` — hide bars for fullscreen
- `@capacitor/splash-screen` — splash control
- `@capacitor/assets` (dev) — generate icon/splash assets from source art

### Config
- `capacitor.config.ts`:
  - `appId: "com.duinoy.game"`
  - `appName: "29"`
  - `webDir: "dist"`
  - splash config (background color, fade)
- `android/` native project generated by `npx cap add android` (committed).

### Build flow
```
npm run build          # tsc --noEmit && vite build  -> dist/
npx cap sync android   # copy dist/ into android/ + sync plugins
cd android && ./gradlew assembleDebug   # -> app/build/outputs/apk/debug/app-debug.apk
```
Documented in README; optionally an npm script `build:android`.

### Platform isolation — `src/platform/native.ts`

Single module wrapping every Capacitor call behind a thin, platform-agnostic API. Phaser
scenes and the engine never import Capacitor directly.

Public API (names indicative):
- `isNative(): boolean` — `Capacitor.isNativePlatform()`
- `initNative(): Promise<void>` — called once from `main.ts`: lock portrait, hide status
  bar, hide splash when ready. No-op on web.
- `onBackButton(handler): void` — register hardware-back handler. No-op on web.

Behavior contract:
- On **web** (`isNative()` false): every function is a safe no-op; no Capacitor plugin is
  touched. This keeps the web build working and unit-testable.
- On **native**: functions delegate to the corresponding Capacitor plugin.

### Wiring into existing code
- `src/main.ts`: `await initNative()` before/around Phaser game creation.
- Back button: register one handler that asks the active Phaser scene what to do —
  - Table scene → show confirm-quit modal (reuse existing modal pattern); confirm returns
    to Menu.
  - Menu scene → `App.exitApp()`.
  - Implementation: scenes expose a `handleBack()` the handler calls, or the handler
    inspects `game.scene` active key. Plan picks one; keep it in `native.ts` + a small hook
    per scene.

### Assets (icon + splash)
- Source a 1024×1024 icon and a splash source image from existing Kenney/game art (or a
  simple composed image).
- Run `npx capacitor-assets generate` to produce Android densities.
- Commit generated resources under `android/`.

## Setup check

A script (`scripts/check-android-env.mjs` or documented `npx cap doctor` + checks) run
before the Android build that verifies:
- `JAVA_HOME` set and a compatible JDK present.
- Android SDK located (`ANDROID_HOME` / `ANDROID_SDK_ROOT`).
- `cap doctor` passes.

Fails fast with an actionable message listing what's missing. README documents the
one-time local setup (SDK, platform-tools, accepting licenses).

## Error handling

- `native.ts` no-ops on web — no thrown errors when plugins absent.
- Native init wrapped so a plugin failure logs and continues (game still playable even if,
  e.g., status-bar hide fails).
- Build script setup-check exits non-zero with a clear message on missing tooling.

## Testing

- **Unit** (`tests/platform/native.test.ts`): mock `@capacitor/core` + plugins.
  - On web (`isNativePlatform` → false): `initNative` and `onBackButton` touch no plugin.
  - On native (mocked true): `initNative` calls orientation-lock + status-bar-hide +
    splash-hide; `onBackButton` registers with `@capacitor/app`.
- **Manual / device**: build debug APK, sideload to emulator or device; verify:
  - launches, portrait locked (rotating device keeps portrait),
  - fullscreen (no status bar),
  - back button: confirm-quit in Table, exit in Menu,
  - icon + splash show.
- Existing 89 engine/bot/UI tests stay green; `npm run build` clean.
